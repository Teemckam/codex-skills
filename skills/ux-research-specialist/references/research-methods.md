# Research Methods

Use this reference when the task involves method selection, research metrics, task design, hypothesis writing, or synthesis structure.

## Method Selection

Choose the method that best matches the decision:

### User Interviews

Best for:

- Understanding goals, motivation, mental models, and context
- Exploring new problems before designing solutions

Avoid:

- Treating stated preference as equal to real behavior

### Usability Testing

Best for:

- Checking whether users can complete a task
- Finding confusion, friction, and breakdown points

Look at:

- Task success rate
- Time on task
- Error rate
- User satisfaction

Avoid:

- Turning one participant's issue into a universal conclusion without checking for repetition

### Surveys

Best for:

- Quantifying attitudes, preferences, and self-reported behavior
- Validating whether a pattern appears across a larger group

Avoid:

- Using survey results alone to explain why behavior happens

### Card Sorting

Best for:

- Testing information grouping
- Understanding category language and mental models

### Tree Testing

Best for:

- Checking whether navigation structure helps users find content

### Contextual Inquiry

Best for:

- Understanding workflow, environment, interruptions, and real-world constraints

### Field Studies

Best for:

- Observing behavior in the user's natural context

### Diary Studies

Best for:

- Long-term behaviors, habits, recurring pain points, and product usage over time

### A/B Testing

Best for:

- Comparing live variants on a measurable outcome

Avoid:

- Using it before the variants are clear enough to test
- Claiming meaning from weak samples or noisy setups

### First Click Test

Best for:

- Checking whether the first action matches the intended path

### Five Second Test

Best for:

- Measuring first impression, clarity, and message comprehension

### Heatmap Analysis

Best for:

- Spotting attention patterns and interaction hotspots

Avoid:

- Treating attention as comprehension or intent

### Session Recordings

Best for:

- Reviewing friction, hesitation, dead clicks, rage clicks, and navigation loops

Avoid:

- Over-generalizing from a handful of sessions

## User Analysis Artifacts

### Persona

Capture:

- Goals
- Motivation
- Problems
- Behavior
- Experience level

### User Journey Map

Capture:

- Stages
- Touchpoints
- Emotions
- Pain points
- Opportunities

### Customer Journey

Use when the full product relationship matters, not just one screen or task.

## UX Metrics

### Task Success Rate

How many participants complete the task successfully.

### Time on Task

How long the task takes when users attempt it.

### Error Rate

How often users make mistakes, backtrack, or choose the wrong path.

### User Satisfaction

How satisfied users report feeling after the experience or task.

### NPS

Useful for relationship-level sentiment, not detailed interaction diagnosis.

### CSAT

Useful for measuring satisfaction with a specific flow, moment, or touchpoint.

### SUS

Useful for a broad usability benchmark across a product or system.

## Research Questions

Strong questions usually focus on comprehension, friction, behavior, or motivation.

Examples:

- How do users understand this interface?
- Which tasks create the most friction?
- What prevents the user from reaching the goal?
- Where does the user get lost?

## Hypothesis Format

Use:

`If we change [element], users will be able to [action], which will lead to [result].`

## Usability Test Plan Template

Define:

1. Research goal
2. Target users
3. Tasks
4. Success criteria
5. Metrics
6. Research method

## Task Scenario Template

Write scenarios from the user's perspective.

Example:

- Task: Create a new project
- Scenario: "Imagine you need to create a new project for your team."

## Findings Synthesis

After research, structure the synthesis around:

- Key insights
- Patterns
- Pain points
- Opportunities

For each issue, note:

- Evidence
- Frequency
- Severity
- User impact
- Recommended next step

## Interpretation Guardrails

Always separate:

- What users did
- What users said
- What you infer
- What should be tested next
