# SEO Playbook

Use this reference when the task involves technical SEO checks, Core Web Vitals, on-page structure, content optimization, or strategic prioritization.

## Technical SEO Checklist

Review:

- `robots.txt`
- `sitemap.xml`
- Canonical tags
- `hreflang`
- Meta robots directives
- Structured data and schema
- Broken links
- Redirect chains
- Duplicate URLs
- Indexation blockers

Fix the blockers before chasing smaller optimization wins.

## Canonicals And Duplicate Content

Check whether:

- Canonicals point to the preferred URL
- Duplicate pages compete with each other
- Parameter or filtered URLs create index bloat
- Pagination or template duplication weakens relevance

## Redirect Hygiene

Look for:

- Redirect chains
- Unnecessary temporary redirects
- Broken redirected paths
- Internal links that still point to old URLs

## Core Web Vitals

### LCP

Reflects:

- Perceived load speed of the main content

Often improved by:

- Faster assets
- Better image delivery
- Better server response
- Less render-blocking work

### INP

Reflects:

- Responsiveness of interactions

Often improved by:

- Less main-thread blocking
- Lighter JavaScript
- Faster event handling

### CLS

Reflects:

- Visual stability

Often improved by:

- Reserving space for media and embeds
- Avoiding late layout shifts
- Stabilizing dynamic content

## On-Page SEO

Review:

- Title uniqueness and clarity
- Meta-description usefulness
- One clear `H1`
- Logical heading hierarchy
- Descriptive image alt text
- Internal-link relevance

Good on-page SEO improves both SERP presentation and on-page comprehension.

## Content SEO

Check:

- Search intent fit
- Topic depth
- Structure and scannability
- Semantic coverage
- Relevance of examples and proof

Use:

- Topic clusters
- Content hubs
- Semantic SEO

Avoid:

- Keyword stuffing
- Thin pages created only for search engines

## Keyword Strategy

Map:

- Primary keywords
- Secondary keywords
- Long-tail keywords
- Search intent

Evaluate:

- Relevance
- Competition
- Organic potential

Keywords should shape structure and targeting, not make the copy unnatural.

## Internal Linking

Use internal links to:

- Surface important pages
- Clarify hierarchy
- Pass contextual relevance
- Improve crawl paths

Good patterns:

- Contextual links
- Breadcrumbs
- Hub-and-spoke content structure

## Structured Data

Use when relevant:

- `Article`
- `Product`
- `Organization`
- `FAQ`
- `BreadcrumbList`

Only add schema that accurately reflects the page content.

## Local SEO

If local intent matters, review:

- Google Business Profile
- NAP consistency
- Local landing pages
- Local keyword targeting

## Mobile SEO

Review:

- Responsive rendering
- Mobile navigation
- Readability
- Tap targets
- Mobile performance

Mobile SEO problems often combine performance, usability, and content-priority issues.

## SEO Analytics

Use analytics and SEO data to find:

- Pages with high impressions but weak CTR
- Pages ranking just below strong positions
- Pages with traffic but poor engagement or conversion support
- Pages with indexing or visibility gaps

## Prioritization Logic

Prefer changes that:

1. Unblock crawling or indexation
2. Improve key page templates
3. Strengthen important commercial or strategic pages
4. Improve Core Web Vitals and mobile UX
5. Expand content authority through clusters and hubs

## Safety Rules

Avoid:

- Cloaking
- Hidden text
- Link schemes
- Thin doorway pages
- AI-generated filler content with no user value
- Manipulative structured data

Use sustainable, guideline-aligned SEO only.
