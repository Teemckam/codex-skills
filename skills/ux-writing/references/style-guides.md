# Style Guides

Use this reference when the task needs stronger reasoning, tone calibration, or a quality check against established UX writing guidance.

## Source Map

### Nielsen Norman Group

Best for:

- Clarity
- scannability
- task-focused wording
- usability reasoning

Use when:

- A message feels confusing
- Copy is too abstract
- You need to justify simplifying wording

### Google UX Writing Guidelines

Best for:

- clear product language
- direct action wording
- concise interface patterns

Use when:

- Writing buttons
- tightening helper text
- simplifying onboarding or system messages

### Microsoft Style Guide

Best for:

- plain language
- consistency
- inclusive, instructional product tone

Use when:

- Writing error messages
- writing guidance text
- reducing ambiguity in system UI

### Mailchimp Content Style Guide

Best for:

- friendly tone
- conversational clarity
- balancing warmth with precision

Use when:

- Text feels cold or robotic
- You need a more human product voice
- Empty states or onboarding need warmth without fluff

## Quick Checks

Before finalizing copy, ask:

- Is it clear on first read?
- Is it short enough to scan?
- Does it sound like a product helping a user?
- Does it tell the user what to do next?
