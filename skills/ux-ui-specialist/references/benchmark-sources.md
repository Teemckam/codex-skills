# Benchmark Sources

Use this reference when the task involves UI inspiration, UX pattern research, trend analysis, or concrete examples.

## Rule

Do not answer with generic advice alone. Prefer named products, real websites, organizations, or specific articles whenever the user asks for inspiration, patterns, trends, or concept work.

## Source Selection

### Mobbin

Best for:

- Real product patterns
- Mobile app flows
- SaaS dashboards
- Onboarding and checkout
- Forms, settings, paywalls, and empty states

Extract:

- Screen sequence
- Interaction pattern
- Information density
- CTA placement
- Form structure
- Repeated product conventions

Avoid:

- Treating polished screenshots as complete UX proof without considering the surrounding flow

### Awwwards

Best for:

- Premium landing pages
- Brand-forward UI
- Motion systems
- Experimental composition
- Storytelling-driven sections

Extract:

- Hero structure
- Narrative pacing
- Transition ideas
- Art direction
- Ways to create memorability

Avoid:

- Copying spectacle that hurts clarity, speed, or accessibility

### Dribbble

Best for:

- Fast visual scanning
- UI motifs
- Color and typography ideas
- Cards, widgets, tables, and component styling

Extract:

- Surface treatment
- Component shape language
- Visual hierarchy ideas
- Micro-layout patterns

Avoid:

- Assuming a shot represents a complete product flow
- Reusing decorative patterns that hurt usability

### Behance

Best for:

- Case studies
- Full design systems
- Brand and product storytelling
- Multi-screen explorations

Extract:

- Design rationale
- System consistency
- Cross-screen logic
- Presentation structure

Avoid:

- Mistaking portfolio storytelling for validated product performance

## UX Research Sources

Use these when the user asks for UX patterns, proven practices, or evidence behind a recommendation.

### NNGroup

Best for:

- Heuristics and usability principles
- Navigation and information architecture
- Forms, content, and interaction patterns
- Research-backed reasoning

Extract:

- Principle name
- User behavior insight
- Tradeoff behind the advice
- Specific interface implication

Avoid:

- Repeating the principle without translating it into the current screen or flow

### Baymard Institute

Best for:

- Ecommerce and checkout patterns
- Search, filters, product lists, and navigation
- Mobile usability and conversion friction
- Interaction details validated at scale

Extract:

- Tested pain point
- Recommended pattern
- Conditions where it matters most
- Conversion or usability implication

Avoid:

- Applying commerce-specific findings too broadly outside their context

### UX Collective

Best for:

- Current practitioner discussions
- Pattern framing and emerging perspectives
- Useful language for explaining trends
- Broader design conversations

Extract:

- Framing of the design problem
- Pattern vocabulary
- Modern context or trend signal
- Ideas worth validating further

Avoid:

- Treating opinion pieces as equal to formal usability research

## Recommended Output

When asked for references, use a compact structure:

1. Goal or interface category
2. 3 to 5 named references
3. Pattern to borrow from each
4. Modern trend signal or UX principle from each
5. Risk or caveat for each
6. Final synthesis

If the answer combines visual and UX sources, label them directly:

- Visual benchmarks
- UX evidence

## Example Synthesis Prompts

- Find 5 real references for a finance dashboard and extract the card, filter, and KPI patterns worth reusing.
- Study recent Awwwards winners for hero layout, transitions, and storytelling, then convert that into a usable landing page direction.
- Compare Dribbble, Behance, and Mobbin references for onboarding and explain which ideas are aesthetic-only versus production-ready.
- Compare NNGroup, UX Collective, and Baymard guidance for checkout friction and recommend one final pattern direction.
