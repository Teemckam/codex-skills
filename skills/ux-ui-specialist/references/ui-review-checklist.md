# UI Review Checklist

Use this reference when the request calls for a structured audit or when a screen needs more than a quick critique.

## Quick Pass

Check these first:

- Is the page purpose obvious in under 5 seconds?
- Is the primary action visually dominant?
- Are labels specific instead of generic?
- Is the information grouped in a way that matches user intent?
- Can the user recover from errors without confusion?

## Hierarchy

- Title and supporting copy establish context fast
- Primary action is easy to find
- Secondary actions do not compete with the main path
- Visual weight matches importance
- Dense screens are broken into digestible sections

## Contrast

- Important actions stand out from surrounding content
- Text contrast is strong enough for fast reading
- UI states remain distinguishable without subtle color differences alone
- Surfaces, borders, and dividers do not blur together

## Readability

- Body copy is easy to scan without visual fatigue
- Labels and helper text stay readable at smaller sizes
- Important information does not rely on tiny, low-contrast text
- Reading rhythm supports both short scanning and deeper review

## Typography

- Text sizes support scanning before deep reading
- Line length feels comfortable, especially on desktop
- Headings, labels, and body copy are clearly differentiated
- Dense copy is chunked into digestible blocks
- Type scale feels intentional rather than random
- Font weight and size support hierarchy clearly

## UX Flow

- The screen makes the next action obvious
- The sequence of steps feels natural
- Supporting information appears before commitment is required
- Users do not need to stop and interpret the interface too often

## CTA

- Primary CTA is clear, visible, and specific
- CTA label matches the user's expected outcome
- Secondary actions do not overpower the main conversion path
- CTA placement aligns with the point of highest intent

## Grid System

- Columns or content blocks align consistently
- Section widths feel intentional rather than arbitrary
- Cards, forms, and content groups share a predictable structure
- Layout rhythm holds up across breakpoints

## Spacing

- Related elements sit close enough to feel grouped
- Separate groups have enough distance to read as distinct
- Vertical rhythm is consistent across sections
- Tight areas feel intentionally dense, not cramped

## Forms

- Labels stay visible and are unambiguous
- Required vs optional fields are clear
- Validation appears at the right moment
- Error messages explain how to fix the problem
- Input types match the data requested
- Long forms are chunked into logical groups

## Navigation

- Users can predict where each option will lead
- Current location is obvious
- Back and close actions are safe and understandable
- Information architecture uses familiar categories
- Important destinations are reachable without hunting

## States

- Empty states explain what to do next
- Loading states reduce uncertainty
- Success states confirm completion and next steps
- Error states preserve context and offer recovery
- Disabled states explain why action is unavailable

## Mobile-First

- The main task is understandable within the first one or two mobile viewports
- Content order still makes sense when stacked into one column
- Primary CTA stays easy to find without relying on desktop layout tricks
- Secondary content can collapse without hurting task completion
- Tables, filters, and dense controls have mobile-safe alternatives
- Tap targets, spacing, and text size support quick touch interaction

## Adaptivity

- The layout scales cleanly from mobile to tablet to desktop
- Key actions remain visible across breakpoints
- Grid changes feel intentional rather than patched
- Content density adjusts without breaking hierarchy
- Components preserve usability when resized or reorganized

## Accessibility

- Color contrast is likely sufficient
- Tap and click targets are comfortably sized
- Interaction does not rely on color alone
- Focus order should make sense by keyboard
- Controls have visible labels or accessible names
- Motion and animation would not block comprehension

## Trust and Tone

- Copy is specific, calm, and human
- Risky actions are clearly signposted
- Pricing, permissions, or irreversible outcomes are transparent
- Tone matches the product and the user's emotional state

## Conversion and Friction

- The screen removes doubts before asking for commitment
- Benefits are visible near the call to action
- Optional complexity is deferred
- Interruptions are minimized
- The user always knows the next step
